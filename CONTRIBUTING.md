# Contributing to Saayam DevSecOps

Thank you for your interest in contributing to the Saayam DevSecOps repository.

## How to Contribute

1. Fork the repository
2. Clone your fork locally
3. Create a new branch
4. Make your changes
5. Commit your changes
6. Push your branch
7. Open a Pull Request

## Contribution Guidelines

Please follow these guidelines when contributing:

- Keep changes focused and small
- Write clear commit messages
- Document your changes if necessary
- Ensure DevSecOps scripts and workflows remain functional

## Areas You Can Contribute

Contributions are welcome in areas such as:

- CI/CD pipeline improvements
- Security automation
- Infrastructure as Code
- Monitoring and logging improvements
- Documentation improvements

## Submitting a Pull Request

1. Push your branch to your fork
2. Open a Pull Request to the main repository
3. Provide a clear description of the change

Thank you for helping improve the Saayam DevSecOps infrastructure.
